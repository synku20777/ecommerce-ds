01 Foundations

The system uses a strict 3-tier token model. Raw SCSS values feed semantic CSS custom properties, and component-scoped tokens reference those semantic roles.

Tier 0

Raw

Brand-agnostic SCSS scales for color, type, spacing, shadows, and breakpoints.

Tier 1

Semantic

CSS custom properties on :root, including dark-theme overrides.

Tier 2

Component

Button, input, toggle, and surface tokens that reference Tier 1 roles.

Token flow 
$raw-brand-500: #3b82f6;
--color-bg-accent: #{$raw-brand-500};
--button-default-bg: var(--color-bg-accent);

.ds-button--default {
  background-color: var(--button-default-bg);
}
Why this matters

Components never consume raw values directly. Theme changes happen by remapping semantic values, while component CSS keeps the same public contract.

Layout

Container width, page gutter, and grid gap.

Layout tokens 
--layout-container-max: 1280px
--layout-gutter-page:   1.5rem  /* 24px */
--layout-gap-grid:      1rem    /* 16px */
Usage

The container max constrains page width at the widest breakpoint. The page gutter sets horizontal padding on the main content area. The grid gap spaces column and row tracks in multi-column layouts.

08 Usage

The demo uses native ES modules and a Sass-only build step. There is no JavaScript bundler requirement for this MVP.

Page setup 
<link rel="stylesheet" href="/dist/tokens.css">
<script type="module" src="/components/index.js"></script>
Build commands 
npm.cmd install
npm.cmd run build
npm.cmd run demo

Theme control

Dark mode is a semantic-token override on html[data-theme="dark"]. Component styles keep referencing the same Tier 2 tokens.
